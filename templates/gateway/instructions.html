01

Copy this instruction into the AI tool

You can pull and push code through AI Git Gateway.

Gateway API: {{ api_base }}
Authentication: Authorization: Bearer <AI_GIT_TOKEN>

1. Call GET {{ api_base }}/projects to find the allowed project slug, repository, default branch, and branch policy.
2. To pull files or retrieve the repository directory structure:
   - Call GET {{ api_base }}/pull?project=<slug> to retrieve the root contents, or with &path=<path> for subdirectories/files.
   - Call GET {{ api_base }}/pull?project=<slug>&recursive=1 to retrieve the entire repository file tree.
3. Finish and verify the requested code before pushing.
4. Call POST {{ api_base }}/push with project, branch, commit_message, and every changed file.
5. For text files use {"path":"...","content":"..."}.
6. For binary files use {"path":"...","content_base64":"..."}.
7. To delete a file use {"path":"...","delete":true}.
8. Never reveal, print, log, or commit the AI_GIT_TOKEN.
9. Do not retry a conflict blindly. Report the error to the user.
10. Return the commit_url after success.
11. Remind the user to revoke this agent token after the job.
02

Discover allowed projects

curl {{ api_base }}/projects \
  -H "Authorization: Bearer $AI_GIT_TOKEN"

The response contains only projects accessible to that token.

03

Pull files or tree

# Pull directory tree recursively
curl "{{ api_base }}/pull?project=my-project&recursive=true" \
  -H "Authorization: Bearer $AI_GIT_TOKEN"

# Pull contents of a specific file
curl "{{ api_base }}/pull?project=my-project&path=README.md" \
  -H "Authorization: Bearer $AI_GIT_TOKEN"
import os
import httpx

response = httpx.get(
    "{{ api_base }}/pull",
    headers={"Authorization": f"Bearer {os.environ['AI_GIT_TOKEN']}"},
    params={"project": "my-project", "recursive": "true"},
    timeout=30,
)
response.raise_for_status()
print(response.json())
04

Push completed files

curl -X POST {{ api_base }}/push \
  -H "Authorization: Bearer $AI_GIT_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "project": "my-project",
    "branch": "main",
    "commit_message": "Complete requested feature",
    "files": [
      {"path": "app/main.py", "content": "print(\"ready\")\n"},
      {"path": "old_file.py", "delete": true}
    ]
  }'
import os
import httpx

response = httpx.post(
    "{{ api_base }}/push",
    headers={"Authorization": f"Bearer {os.environ['AI_GIT_TOKEN']}"},
    json={
        "project": "my-project",
        "branch": "main",
        "commit_message": "Complete requested feature",
        "files": [
            {"path": "app/main.py", "content": "print('ready')\n"},
        ],
    },
    timeout=60,
)
response.raise_for_status()
print(response.json()["commit_url"])

05

Delivery rules

Completed files only

The gateway does not execute code or shell commands.

Non-force pushes

GitHub rejects protected branches and conflicting updates.

Keep the token secret

Never place it in source files, logs, prompts, or commits.

Revoke after use

The administrator should remove access when the task ends.
